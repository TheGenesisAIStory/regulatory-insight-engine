# Fiorell.IA v0.1.0-beta

Status: experimental beta package for the Fiorell.IA product/spec layer.

## What It Is

Fiorell.IA is an Italian-first banking regulatory assistant layer built on top of the shared offline RAG runtime in this repository.

It defines product identity, behavior expectations, prompt specs, evaluation criteria and beta release messaging. It does not own serving, retrieval, indexing, generation or runtime configuration.

## Behavior Goal

Fiorell.IA should answer only when local retrieved sources support the response. It should refuse clearly when a question is outside banking regulation or not sufficiently documented in the local corpus.

## Strongest Current Coverage

The current corpus is strongest around:

- default;
- internal controls;
- own funds;
- prudential supervision.

## Not Claimed Yet

This beta does not claim complete coverage of:

- all IFRS 9;
- all Pillar 3;
- the complete EBA/Basel consolidated perimeter;
- all Italian bank disclosures.

## Included

- Product README and architecture boundary.
- Behavior specification and v0 scope.
- Prompt and refusal templates.
- Seed Fiorell.IA eval set and rubric.
- Evaluation plan and go/no-go rules for controlled beta validation.
- Training experiment manifest.
- Release checklist and public positioning notes.

## Limitations

- Experimental beta, not production-ready.
- Not legal, accounting, supervisory or regulatory advice.
- Requires local corpus coverage and a valid shared runtime index.
- Bank-specific answers require the relevant local disclosure source.
- Any model specialization must be evaluated against the shared RAG baseline before use.

## Runtime Boundary

Fiorell.IA does not define thresholds, domain gate modes, retrieval parameters, model-serving settings or API behavior. Those remain owned by the shared runtime.

## Release Decision

Publishable as a beta product/spec package only. Do not present as production-ready or complete regulatory coverage.
